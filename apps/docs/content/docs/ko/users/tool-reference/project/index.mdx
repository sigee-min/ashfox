프로젝트 도구는 세션의 바깥 경계를 관리합니다. 프로젝트를 준비하거나 재사용하고, 현재 상태를 조회하고, 검증과 프리뷰를 거쳐 최종 산출물을 내보내는 단계가 이 카테고리에 포함됩니다.

대부분의 클라이언트에서는 이 도구들을 미세한 편집마다 호출하기보다 단계 전환 시점에 호출합니다. 이렇게 구성하면 상태가 명확해지고 실패 원인을 추적하기가 쉬워집니다.
