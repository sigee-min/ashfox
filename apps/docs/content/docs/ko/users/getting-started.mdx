---
title: 시작
description: Ashfox MCP 도구를 연결하고 검증된 내보내기까지 완료하는 첫 실행 흐름.
---

시작 단계의 목표는 단순히 한 번 호출이 성공하는 것이 아니라, 연결 확인부터 변경 적용, 검증, 내보내기까지 끊기지 않는 작업 루프를 만드는 것입니다. 이 루프가 안정적으로 돌아가면 이후 모델 규모가 커져도 같은 방식으로 운영할 수 있습니다.

먼저 Blockbench Desktop에서 Ashfox 플러그인을 실행하고 MCP 클라이언트를 `http://127.0.0.1:8787/mcp`에 연결합니다. 연결 직후 `list_capabilities`를 호출해 현재 서버 스키마를 읽으면, 클라이언트가 어떤 입력 형식을 따라야 하는지 기준점을 확보할 수 있습니다.

이후 첫 세션 흐름은 `ensure_project`로 프로젝트 컨텍스트를 확보하고, `get_project_state`로 현재 상태와 revision을 확인한 다음, 변경 도구 호출에 `ifRevision`을 전달해 충돌을 방지하는 순서로 진행합니다. 변경이 끝나면 `validate`로 구조적 오류를 먼저 점검하고, `render_preview`로 시각 결과를 확인한 뒤 `export`로 마무리하면 됩니다.

정상 환경에서는 각 단계의 응답이 다음 단계 입력으로 자연스럽게 이어져야 하며, 중간에 수동 복구가 필요하지 않아야 합니다. 특정 단계에서 실패하면 문제 해결 섹션에서 실패 유형에 맞는 문서를 바로 찾아 원인을 좁히는 방식이 가장 빠릅니다.
