---
title: 학습 경로
description: 설치, 워크플로, 실행 문서를 연결해서 읽는 권장 순서.
---

문서가 태스크 중심으로 느껴질 때는 이 페이지를 기준으로 읽으면 흐름을 잡기 쉽습니다. 핵심은 바로 실행하는 것보다 먼저 단계의 목적과 경계를 이해하는 것입니다.

먼저 설치와 시작 문서로 실행 환경을 안정화하고, 이후에는 Blockbench Plugin 가이드 또는 Ashfox 가이드 중 현재 작업 맥락에 맞는 트랙을 선택하세요. 트랙 가이드로 전체 흐름을 이해한 다음, 도구 레퍼런스에서 필드 단위 세부 계약을 확인하고 문제 해결 문서를 병행하면 시행착오를 크게 줄일 수 있습니다.

이 순서는 가이드를 우선하고 태스크를 보조로 쓰는 구조입니다. 작업 플레이북은 빠른 실행에 강하지만, 가이드 맥락을 먼저 이해했을 때 품질이 가장 안정적입니다.
