## Overview
- Tool name: `export`
- Category: Project
- Strict schema: `true`

## Tool Description
Exports the current project to the requested output format.

## Input Schema (JSON)
```json
{
  "type": "object",
  "required": [
    "format",
    "destPath"
  ],
  "additionalProperties": false,
  "properties": {
    "format": {
      "enum": [
        "java_block_item_json",
        "gecko_geo_anim",
        "animated_java",
        "generic_model_json",
        "gltf",
        "auto"
      ]
    },
    "destPath": {
      "type": "string"
    },
    "includeState": {
      "type": "boolean"
    },
    "ifRevision": {
      "type": "string"
    }
  }
}
```

## Field Reference
| Field | Type | Required | Description | Constraints |
| --- | --- | --- | --- | --- |
| `format` | `enum(...)` | ✅ | Export target selector. | `java_block_item_json`, `gecko_geo_anim`, `animated_java`, `generic_model_json`, `gltf`, `auto` |
| `destPath` | `string` | ✅ | Destination file path. | For `gltf`, use `.gltf` or `.glb` |
| `includeState` | `boolean` | ❌ | Include state snapshot in MCP response. | - |
| `ifRevision` | `string` | ❌ | Optimistic revision guard. | Must match current revision |

## Minimal Request Example
```json
{
  "format": "gltf",
  "destPath": "./out/model.glb"
}
```

## Auto Mode
- `format: "auto"` resolves by `destPath` extension first.
- `.gltf` or `.glb` resolves to native glTF export.
- Otherwise it falls back to the active project format default export target.
- If no rule matches, the tool fails with `invalid_payload`.

## Response Example
```json
{
  "structuredContent": {
    "path": "value"
  }
}
```

## Operational Notes
- Use `ifRevision` for mutation calls when available.
- Do not send fields outside the declared schema.
- Run `validate` before `export` for release safety.
- Use `generic_model_json` for Generic Model (`free`) projects when deterministic JSON output is required.
